    Cloud Native Release Engineering

    SLO validation, A/B(/n) testing with business metrics, chaos injection, and progressive rollouts.

    Quick start in 5 mins

    Features

    Declarative Experiments

    Iter8 experiments enable SLO validation, A/B(/n) testing for optimizing business reward metrics, chaos testing, progressive rollouts with advanced traffic engineering, and their hybrids.

    Diverse App Frameworks

    Experiment with Kubernetes deployments, Helm releases, Knative (serverless) services, KFServing (ML) inference services, Seldon (ML) deployments, and more.

    Builtin/Custom Metrics

    Iter8 can generate HTTP GET/POST requests and collect latency and error related metrics. Experiments can also use custom metrics from any DB like Prometheus, Elastic, NewRelic, and Sysdig.

    Traffic Engineering

    Experiments can use any feature provided by the ingress or service mesh in your cluster like traffic mirroring, splitting traffic splitting based on percentage, headers, or cookies, and session affinity.

    CI/CD/GitOps

    Experiments can send HTTP notifications. They can also automatically create git pull requests. Embed them within any CI/CD/GitOps process for promoting new app versions.

    AI-driven & Statistically Robust

    Iter8 uses statistically rigorous Bayesian analysis for version assessment and multi-armed bandit algorithms for progressive traffic shifting.
